## 新增需求

### 需求:用户可在制作端预览翻页阅读体验

项目工作台必须提供预览 Tab 或模式，使用 Pixi 渲染当前书页；用户必须能切换上一页/下一页并看到页码指示。

#### 场景:翻页预览

- **当** 用户在预览模式点击「下一页」
- **那么** 画布必须切换至下一页场景内容，且页码指示必须更新

### 需求:预览引擎必须支持 L1 动效模板

预览必须能读取当前页 `scene.json`（mock）并执行至少一种 L1 模板；系统必须实现 `stagger_pop_in` 模板：用户点击画布后，指定图层必须按配置延迟依次播放 pop_in 动效。

#### 场景:三只小猪依次弹出

- **当** 当前页 scene 使用 `stagger_pop_in` 且用户点击画布
- **那么** 多个占位图层必须按 sequence 中 delay 依次显现，且不得自动无交互播放完毕

### 需求:预览引擎必须支持 L2 帧序列占位播放

预览必须支持 `sprite_sequence` 类型图层，按 `fps` 与 `loop` 配置循环播放 mock 帧 URL 列表；本阶段必须使用占位图或内置 mock 资产，不得依赖真实 AI 逐帧生成。

#### 场景:播放施工动画帧序列

- **当** 当前页 scene 含小黑猪挥锤序列且用户进入预览
- **那么** 画布必须在背景层之上播放帧序列动画，且角色图层必须与背景对齐显示

### 需求:预览数据必须来自 Scene Graph 结构

预览引擎禁止硬编码单页布局；必须从 `scene.json` 读取 `layers`、`template`、`trigger`、`sequence` 字段驱动渲染，且类型定义必须来自 `scene-schema` 包。

#### 场景:切换页面加载新 scene

- **当** 用户从介绍页预览切至盖房页
- **那么** 引擎必须卸载旧场景并按新页 `scene.json` 重建图层与交互，不得残留上一页图层
