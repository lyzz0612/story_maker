## 新增需求

### 需求:用户可浏览与创建绘本项目

系统必须提供项目列表页，展示已有绘本项目（标题、更新时间、页数、画风）；用户必须能发起新建项目并进入工作台。

#### 场景:查看项目列表

- **当** 用户打开项目列表页
- **那么** 系统必须展示 seed 或已创建的至少一个示例项目

#### 场景:新建项目

- **当** 用户通过新建向导提交简略梗概、所选主角、画风、LLM 与生图配置
- **那么** 系统必须创建新项目并跳转至该项目工作台

### 需求:用户可通过简略梗概获得模拟分页大纲

新建项目后，系统必须通过 mock 逻辑生成结构化分页大纲；每页必须包含页码、摘要、页文案占位、出场固定角色与临时角色标注；默认页数必须为 6～8 页。

#### 场景:生成三只小猪示例大纲

- **当** 用户输入含「三只小猪」语义的梗概并创建项目
- **那么** 大纲必须包含不少于 6 页，且至少一页标注临时角色，至少一页可标注家庭主角出场

### 需求:工作台必须提供大纲面板与对话侧栏联动

项目工作台必须同时展示分页大纲面板与对话输入区；用户发送消息后，系统必须通过 mock 规则返回 AI 回复文本，并在匹配时更新大纲指定页内容。

#### 场景:对话修改指定页

- **当** 用户在对话中输入「第 3 页让爸爸出场」
- **那么** 系统必须在对话区显示确认性 mock 回复，且第 3 页大纲必须反映爸爸出场变更

#### 场景:对话修改语气

- **当** 用户输入「结尾不要太 scary」
- **那么** 系统必须更新末页摘要或文案占位，且对话区必须有对应回复

### 需求:创作流程 UI 必须参考竞品工作台形态

工作台必须采用多面板布局（侧栏导航 + 中央主区 + 右栏对话/属性），中央主区必须支持按分页浏览大纲卡片；禁止将完整创作流程降级为仅单页表单向导。

#### 场景:进入项目工作台

- **当** 用户打开某绘本项目
- **那么** 界面必须同时可见大纲分页结构与对话入口，且不得要求用户离开本页才能完成大纲查看
