## 新增需求

### 需求:用户可逐页查看与编辑分页内容

项目工作台必须提供分页编辑视图；每页必须展示页码、摘要、正文文案、配图状态、出场角色信息；用户必须能编辑页文案并保存至 mock 存储。

#### 场景:编辑页文案

- **当** 用户修改第 2 页正文并保存
- **那么** 重新加载该页时必须显示更新后的文案

#### 场景:查看页结构约束

- **当** 用户打开任意分页详情
- **那么** 界面必须体现「一页一图一文」结构，展示配图区域与文案区域

### 需求:用户可触发模拟配图生成

每页必须提供「模拟生成配图」操作；触发后页状态必须经 `generating` 过渡至 `ready`，并填充占位配图 URL 与 mock `scene.json` 路径。

#### 场景:模拟生成单页配图

- **当** 用户对第 4 页点击「模拟生成配图」
- **那么** 页状态必须短暂显示生成中，随后展示占位图，且不得调用真实生图 API

### 需求:分页必须关联项目级画风与模型选择

分页 mock 生成时必须读取项目已锁定的 `artStyleId` 与默认/项目指定的生图配置 id，并在页元数据中可查看（只读展示即可）。

#### 场景:查看页生成上下文

- **当** 用户打开已创建项目的某一页
- **那么** 页详情或侧栏必须显示本书锁定画风名称
