## 新增需求

### 需求:用户可在页级编辑资产区域

系统必须为每一页提供资产区域编辑视图；用户必须能加载页级图集（mock 图片 URL），在画布上拖拽绘制或调整矩形区域，并为每个区域指定用途类型。

#### 场景:框选背景区域

- **当** 用户在资产编辑视图拖拽矩形覆盖图集上部背景区并标记为 `background`
- **那么** 系统必须保存归一化坐标 `{ x, y, w, h }`（取值 0～1）及 role 至该页 `assetSheet.regions`

#### 场景:框选动画帧区域

- **当** 用户为图集中部四个相邻区域分别标记为 `sprite_frame` 并归入同一 `sequenceId`
- **那么** 系统必须按 `frameIndex` 顺序保存帧列表，并支持设置 fps 与 loop 标志

### 需求:静态背景区域不得与角色参考合并

资产区域类型为 `background` 的区域必须表示无角色场景层；角色参考图必须来自家庭角色库单图引用，禁止在页级图集中定义 `character_ref` 类型区域。

#### 场景:仅标记背景与帧

- **当** 用户完成小黑猪盖房页的区域标注
- **那么** 必须存在至少一个 `background` 区域与至少两个 `sprite_frame` 区域，且不得出现角色参考 turnaround 区域类型

### 需求:合成预览区域不得导入运行时

系统可支持 `preview_only` 区域类型用于对照站位；该区域必须仅用于制作端参考展示，预览引擎播放时不得将其作为运行时图层导入。

#### 场景:预览忽略 preview_only

- **当** 用户在工作台预览当前页且资产表含 `preview_only` 区域
- **那么** Pixi 预览必须仅合成 `background` 与动画序列帧，不得渲染 `preview_only` 裁剪结果

### 需求:区域裁剪导出本阶段可为模拟

本阶段系统允许「导出裁剪」仅更新 mock 元数据或控制台日志；禁止要求真实文件写入磁盘，但 JSON 结构必须与未来文件导出兼容。

#### 场景:模拟导出裁剪

- **当** 用户点击「导出区域」
- **那么** 系统必须根据 regions 输出每区域对应的逻辑文件名映射，且不得失败于缺少后端文件系统
