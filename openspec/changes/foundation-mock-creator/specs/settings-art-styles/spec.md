## 新增需求

### 需求:用户可管理画风预设

系统必须允许用户对画风预设执行增删改查；每条画风必须包含名称、描述、prompt 后缀片段（`promptSuffix`），并可带预览占位图 URL。

#### 场景:新增画风

- **当** 用户在画风管理页填写名称与 prompt 片段并保存
- **那么** 新风画必须出现在画风列表，且新建故事时可被选择

#### 场景:编辑画风 prompt

- **当** 用户修改某画风的 `promptSuffix` 并保存
- **那么** 更新后的 prompt 片段必须在后续读取时生效

### 需求:系统必须提供初始画风种子数据

系统启动时必须预置不少于 3 条示例画风（如：温暖水彩、柔和粉彩、绘本铅笔），以便新建项目时即可选择。

#### 场景:首次打开设置页

- **当** 用户首次进入画风管理页且未自定义任何画风
- **那么** 列表必须展示 seed 预置的画风条目

### 需求:单本书籍锁定画风

新建故事时必须选择一种画风；项目创建后必须将 `artStyleId` 写入项目元数据，且在项目工作台中可查看当前锁定画风（本阶段禁止无提示切换画风重绘）。

#### 场景:创建项目时选择画风

- **当** 用户在新建向导中选择「温暖水彩」并完成创建
- **那么** 项目详情必须显示已锁定画风为「温暖水彩」
