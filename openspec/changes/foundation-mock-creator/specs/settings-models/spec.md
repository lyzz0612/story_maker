## 新增需求

### 需求:用户可管理故事 LLM 配置

系统必须允许用户查看、新增、编辑、删除故事 LLM 提供方配置；每条配置必须包含名称、provider 类型、baseUrl、apiKey、model 字段；必须支持将一条配置设为默认 LLM。

#### 场景:新增 LLM 配置

- **当** 用户在设置页填写有效的 LLM 配置并保存
- **那么** 配置必须出现在列表中，且可通过 mock API 读取

#### 场景:设置默认 LLM

- **当** 用户将某条 LLM 配置设为默认
- **那么** 该配置必须标记为 `isDefault: true`，且其他 LLM 配置的 `isDefault` 必须为 false

### 需求:用户可管理生图模型配置

系统必须允许用户查看、新增、编辑、删除生图模型提供方配置；每条配置必须包含名称、provider 类型、baseUrl、apiKey、model 字段；必须支持将一条配置设为默认生图模型。

#### 场景:新增生图配置

- **当** 用户在设置页填写有效的生图配置并保存
- **那么** 配置必须出现在列表中，且新建故事向导必须能选择该配置

#### 场景:禁止删除唯一默认生图配置

- **当** 用户尝试删除当前唯一的默认生图配置且未先指定新默认项
- **那么** 系统必须阻止删除并给出明确提示

### 需求:模型配置使用模拟存储

本阶段系统禁止调用外部真实 LLM 或生图 API；所有配置变更必须通过 mock API 持久于服务端内存（或约定的本地镜像），并在服务重启后可通过 seed 数据恢复演示状态。

#### 场景:保存配置不触发外网请求

- **当** 用户保存任意模型配置
- **那么** 系统不得向外部 AI 服务商发起网络请求
