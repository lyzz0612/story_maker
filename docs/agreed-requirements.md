# 已达成一致事项（Raw）

> 探索阶段结论文档。按讨论顺序与主题整理，供后续 OpenSpec 提案与 MVP 实现引用。  
> 非最终实现规格；标有「待定」「后续」的条目刻意保留开放性。

---

## 1. 产品定位

- 类似 BookletAI、ReadKidz 的**绘本创作工具**，但面向**个人使用**，给自己孩子做书。
- **单用户**，初版不考虑多用户、账号、协作、云同步。
- **制作**在 Story Maker 制作端完成；**阅读**在自研平板 App 中使用。
- 用户曾澄清：「在 App 上用」指**导出物在 App 里阅读**，不是「在 App 里制作」。
- 导出格式与 App 导入契约**后续再定**；功能讨论阶段已确认交互数据需作为一等公民（见第 7 节）。

---

## 2. 与参考产品的关系

### 借鉴

- BookletAI：简略输入 → 大纲 → **对话调整细节**；逐页修订。
- ReadKidz：**角色一致性**、画风预设、分页结构。

### 不照搬

- ReadKidz 级自由画布、贴纸库、视频时间轴、KDP/YouTube 发布。
- BookletAI 的通用小册子 / 打印优先定位（打印可作为次要能力，非核心）。
- 多用户 SaaS 运营形态。

### 差异化（核心亮点）

1. **家庭主角库**：宝宝、爸爸妈妈等跨故事形象一致。
2. **对话式故事打磨**（绑定结构化分页大纲）。
3. **互动编排**：翻页 + 页内轻交互（如点击后小猪依次冒出）——**最主要亮点**。

## 4. 角色体系

### 4.1 两类角色

```
主角团（家庭角色库）     故事临时角色
────────────────────     ────────────────
宝宝、爸爸妈妈等         三只小猪、大灰狼等
跨故事必须同形象         仅当前故事需要
进入全局角色档案         不进入全局库
生图时强一致约束         普通 prompt 即可
```

### 4.2 家庭角色库（功能要求）

- 创建 / 编辑主角：名字、关系、外貌描述。
- **定稿形象**：生成候选或上传 → 选定「标准照」/ 参考图。
- 新建故事时**勾选出场主角**。
- 换形象、换发型等需**显式操作**，避免静默换脸。
- 一致性实现认知：不能只靠 prompt；需参考图 + 固定描述（具体模型能力 MVP 验证）。

### 4.3 临时角色

- 由单本故事大纲引入。
- 同一本书内大致像即可；**不要求**跨书一致。
- 下一本再讲三只小猪，可以重新生成。

### 4.4 待定（探索时提出，未最终拍板）

- 主角形象首次来源：纯 AI 生成 vs 上传真实照片 vs 两者都要。

## 5. 故事创作流程

### 5.1 输入方式

- 用户输入**极简简略文字**即可，不必写全文。
- 示例语义：「三只小猪盖房子，宝宝扮演小猪，温馨一点，8 页左右」。

### 5.2 流程

```
简略文字
    → AI 生成分页大纲（每页摘要 + 出场角色标注）
    → 制作端对话调整细节
    → 定稿后进入配图 / 排版 / 交互编排
```

### 5.3 对话与大纲

- 对话绑定**当前书籍**。
- 需有**大纲面板**与对话联动（改第 N 页时结构可预期更新）。
- 支持改情节、改单页、改语气、调整出场角色等。
- 对话示例：「第 3 页让爸爸出场」「结尾不要太 scary」「第 5 页加一句宝宝台词」。

### 5.4 结构约束（建议，讨论中倾向）

- 初版倾向 **一页 = 一图 + 一段文字**，降低分页与交互复杂度。
- 页数可配置或在大纲阶段约定（如 6～12 页睡前短篇）。

### 5.5 明确不做（初版）

- 网页研究、贴 URL 抓资料（BookletAI 能力，与本项目无关）。

## 6. 视觉与生成

### 6.1 画风

- 预设**多种画风**（数量「很多」，MVP 可先实现子集如 10～20，结构可扩展）。
- 每本书**锁定一种画风**；更换画风视为重绘，需有提示。

### 6.2 生图模型

- 用户可选择**不同生图模型**生成绘本页。
- 画风预设与生图模型为**两个独立维度**（画风 = prompt/美学规则；模型 = 底层引擎）。

### 6.3 出图策略（低成本倾向，讨论共识）

| 策略 | 说明 | MVP 态度 |
|------|------|----------|
| 分层单独生成 | 背景 + 各可交互元素分开生成（透明背景） | **默认推荐** |
| 整图 + 抠图分割 | 省 API 次数，边缘与热点易翻车 | 可选对比试验 |
| 混合 | 氛围页整图；交互页分层 | 务实 |

### 6.4 一致性规则

- **主角**：跨页、跨书强一致（尽力而为，MVP 记录实际成功率）。
- **临时角色**：仅本书内合理即可。

### 6.5 页级图集与区域编辑（生图试验结论）

生图提示词**不应绑定精确像素与切图坐标**；尺寸规划与裁剪由制作端工具负责。

| 原则 | 说明 |
|------|------|
| 生图只管内容 | 画风、角色、动作语义；不要求 AI 输出严格工程网格 |
| 静态背景不含角色 | 运行时背景层留站位，动画帧叠加上去；带角色的「合成预览」仅验收用 |
| 角色一图一库 | 每个角色单独一张参考图（如 `ref_black_pig.png`），**不合并进页级图集**；生图引用用 `characterId → ref.png` |
| 页级图集可合并 | 一张大图可含：无角色背景 + 动画帧序列 + 可选合成预览；**不含**角色设定 turnaround |
| 区域手框为主 | AI 分区排版不稳定（标题烧录、帧宽不等、假透明棋盘格）；导入后工具内拖拽矩形，存归一化坐标 |
| 半自动可选 | 白缝横条猜标题行、棋盘格块猜帧区 — 仅作初框，需人工微调 |

## 7. 阅读体验与交互编排（主亮点）

### 7.1 体验目标

- **翻页效果**（必选）。
- **页内小动画 / 轻交互**（核心亮点）。
- 示例：三只小猪介绍页，**点击后小猪一只只冒出来**（非自动放映幻灯片）。

### 7.2 体验分层（已命名）

| 层级 | 内容 | 定位 |
|------|------|------|
| **L0** | 翻页 + 静态整页图 | 兜底 |
| **L1** | 分层静态图 + 程序动效 + 动效模板库 | **默认主线** |
| **L2** | 少量元素 3～4 帧循环（走、哈气、施工等） | **初步可行**（见 §7.9）；作点缀，非全书默认 |
| **L3** | 极少数页短视频 / 循环视频作背景层 | 非 MVP；整页视频不默认 |

### 7.3 低成本主方案（讨论结论）

**主方案不是「全页生成视频」，也不是默认「全角色序列帧」。**

而是：

```
Page = Scene Graph
  • background + 若干 PNG 图层（可点元素独立）
  • interaction spec（触发器 + 动效序列）
  • 动效由代码模板执行（pop_in、shake、stagger 等）
```

### 7.4 动效模板（初版候选，非封闭列表）

- `stagger_pop_in` — 依次弹出（三只小猪）
- `tap_reveal` — 点击揭示
- `shake` — 摇晃
- `float_idle` — 轻微漂浮
- `slide_in` — 滑入
- `scale_pulse` — 点击强调放大

模板由代码实现；AI/规则负责**选模板、列元素、填顺序与延迟、粗坐标**。

### 7.5 用户设想的其他路线（定位）

| 路线 | 结论 |
|------|------|
| 生成视频 | 贵、难拆元素、难「点一只出一只」；仅可作 L3 极少数页的背景层 |
| 角色动画序列帧 + 程序控制 | 适合 L2 循环动作；不是「冒出来」类交互的主方案 |
| AI 生成任意动画代码 | **不做**；难预览、难维护、有风险 |

### 7.6 交互调整方式（倾向）

- 默认：AI 按模板生成交互方案。
- 对话可改：如「改成点一下才出下一只」。
- 少数页是否需要拖拽微调坐标：**待定**（探索时提到半手动为可选，非 MVP 必需）。

### 7.7 制作端预览 vs 最终 App

- 制作端需能**翻页 + 跑交互**，用于验收。
- 最终 App 体验目标一致；导出载体后续定义，但场景 JSON 结构应可复用。

### 7.8 动画深度

- 初版交互主要指 **点击触发、依次出现、简单 easing**（L1）。
- 连续动作（盖房挥锤、狼哈气等）→ **L2 短序列帧**；MVP 可含 1～2 处样本页，不必全书铺开。

### 7.9 L2 探测结论（2026-06-08，三只小猪 / 小黑猪盖木屋）

**初步判断：L2 可行。**

| 维度 | 观察 |
|------|------|
| 帧间一致性 | 小黑猪 4 帧施工动作（举锤→挥下→接触→检查）角色可辨认、体型稳定 |
| 背景分离 | 无角色静态背景可单独使用；与动画帧叠加逻辑成立 |
| 角色参考 | 单独三视图参考图质量可用；比图集内嵌角色区更易引用 |
| 工程成本 | **不在生图**，在工具：手框区域、去假透明、对齐站位 |

**仍须工具支持（非 AI 能力问题）**

- 图集内中文标题带需裁掉；帧格**非等宽**，不能按固定网格自动切
- 棋盘格为假透明，导出需抠图/键色
- 合成预览仅对照站位，不导入运行时

**产品含义**

- L1 仍为默认主线；L2 作为**可选增强**（施工、哈气等 3～4 帧循环）
- 「冒出来」类交互仍优先 L1 模板（`stagger_pop_in`），不必用 L2 序列帧

## 9. 排版

- 倾向 **固定版式**（上图下文 / 左图右文等 2～3 种），非自由画布。
- 字图位置以模板为主；对话驱动的结构一页一图一文。
- 复杂贴纸、气泡、多文本框：**初版不做**。

## 10. 完善层（配音 / 音乐 / 视频）

- ReadKidz 式配音、BGM、整书转视频：**非初版重点**。
- 文本按页结构化，为将来音频预留字段即可。
- 现场由家长朗读 → 零功能成本，仍成立。

## 12. 功能优先级（讨论归纳）

### P0 — MVP 核心

- 家庭角色库（创建、参考形象、跨书引用）
- 简略输入 → 分页大纲 → 对话修订
- 画风选择 + 生图模型选择
- 分页配图（主角强一致 / 临时角色普通）
- 交互编排（L1 至少一种模板跑通）
- 制作端翻页 + 交互预览
- 本地项目保存

### P1 — 第一本后会想要

- 单页重写文案 / 重绘
- 语气与难度调整
- 复制项目
- 基础撤销
- 对话修改交互参数（触发方式、间隔等）

### P2 — 可晚做

- 上传照片融入故事
- 自己写全文再 AI 分页
- 扉页 / 封底 / 献词
- 简单 TTS 试听
- 导出 PDF（打印次要）

### P3 — 明确先不做

- 自由画布、贴纸库
- 姿势表情精细控制、Chat PS 类局部改图
- 视频时间轴
- 多语言、KDP 上架
- 多用户与云

### MVP 特别探测项

- ~~**L2 短序列帧**是否已可轻松用于 1～2 个动作点~~ → **初步结论：可行**（2026-06-08，《三只小猪》小黑猪盖木屋试验）；配套需页级区域编辑器
