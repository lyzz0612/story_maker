# Story Maker — 技术选型

> 面向 MVP：优先验证 AI 能力与互动编排可行性，控制工程复杂度。单用户、本地优先。

## 选型原则

1. **制作端与阅读端分离**：本仓库聚焦制作端；平板 App 自研，接口以后再定。
2. **本地优先**：无账号、无多租户；项目落盘可备份、可手改。
3. **AI 可插拔**：故事 LLM、生图模型、（可选）抠图/分割均可替换，便于 MVP 对比能力。
4. **交互数据一等公民**：页内容用场景图（Scene Graph）描述，预览与将来导出共用同一套结构。
5. **先跑通再优化**：不 premature 上微服务、不上复杂画布编辑器。

## 推荐技术栈（MVP）

### 制作端应用形态

| 选项 | 建议 | 说明 |
|------|------|------|
| **本地 Web 应用** | ✅ 首选 | 浏览器访问 `localhost`；UI 开发效率高，预览引擎与 Web 技术栈统一 |
| 桌面壳（Tauri / Electron） | 二期可选 | MVP 不必；需要时再套壳 |
| 平板原生创作端 | ❌ 非 MVP | 已明确不在 App 内制作 |

### 前端

| 技术 | 选型 | 理由 |
|------|------|------|
| 语言 | **TypeScript** | 与场景图 JSON、API 契约类型安全 |
| 框架 | **React** | 组件化适合「大纲面板 + 对话 + 预览」多面板布局 |
| 构建 | **Vite** | 轻量、本地开发快 |
| 样式 | **Tailwind CSS** 或 CSS Modules | 按团队习惯；MVP 不追求设计系统 |
| 状态 | **Zustand** 或 React Context | 项目/角色/当前页状态；避免过重 Redux |
| 预览 / 交互引擎 | **Pixi.js** | 2D 分层、精灵、tween、点击命中；比 Phaser 更贴「场景编排」 |
| 动效补间 | **@pixi/tween** 或自写 easing | 模板动效（pop_in、shake 等）在引擎层实现 |

> 备选：若更熟悉 Flutter，可用 Flutter 做制作端 UI + `flame` 做预览；但 AI 编排后台仍建议 Node/Python 一侧，整体不如 Web 统一。

### 后端 / 编排层

| 技术 | 选型 | 理由 |
|------|------|------|
| 运行时 | **Node.js** | 与前端同语言；调用多家 AI HTTP API 方便 |
| 框架 | **Fastify** 或 **Hono** | 轻量 REST；MVP 不需要 Nest 全家桶 |
| 任务队列 | **内存队列**（p-queue） | 单用户；控制生图并发即可 |
| 配置 | **.env + 本地 config 文件** | API Key 存本机，不入库 |

> 备选：**Python FastAPI**。若后续重度依赖本地 CV（抠图、分割、帧处理），可增加 Python  sidecar；MVP 可先 Node 单体。

### 数据存储

| 层级 | 选型 | 说明 |
|------|------|------|
| 元数据 | **SQLite**（better-sqlite3 / Drizzle ORM） | 角色、项目、页、对话线程索引 |
| 资产 | **本地文件系统** | `png`/`webp` 图层、可选 `mp3`、场景 JSON |
| 项目布局 | 每项目一目录 | 便于备份、手动检查、将来打包导出 |

示例目录：

```
data/
  characters/
    {characterId}/
      profile.json
      reference.png
  projects/
    {projectId}/
      meta.json
      outline.json
      chat.jsonl
      pages/
        01/
          scene.json      # 场景图 + 交互编排
          bg.png
          pig1.png
          ...
```

### AI 能力接入（可插拔）

#### 故事 / 大纲 / 对话

| 用途 | MVP 建议 | 集成方式 |
|------|----------|----------|
| 梗概 → 分页大纲 | 任选强指令遵循模型 | OpenAI-compatible API |
| 对话修订大纲 | 同上，带当前大纲上下文 | 流式输出可选 |
| 交互意图识别 | 规则 + LLM 结构化输出 | JSON schema / tool call |

候选：GPT-4o、Claude、DeepSeek 等——MVP 阶段通过适配器抽象，便于 A/B。

#### 图像生成

| 用途 | MVP 建议 | 说明 |
|------|----------|------|
| 分页配图 | **Fal.ai / Replicate** 聚合多模型 | 便于切换 Flux、SDXL 等做能力测试 |
| 主角一致 | 参考图 + 固定 prompt 片段 | 视模型支持可试 IP-Adapter、character reference |
| 分层出图 | 每元素单独生成（透明背景 prompt） | 默认策略，利于交互 |
| 整图抠图（可选） | **rembg** 本地或 API | 备选降本策略，MVP 可作对比试验 |

#### L2 序列帧

| 路径 | 说明 |
|------|------|
| 页级图集 + 手框裁剪 | **已验证可行**（小黑猪 4 帧施工）；背景无角色 + 动画帧同 sheet 或分拆均可 |
| 图生图变体 | 标准图 + 「下一动作帧」prompt，生成 3～4 张 |
| 角色参考 | **一角色一图**，`characterId → ref.png`；不嵌入图集 |
| 供应商 sprite/motion API | 若有则封装为可选后端 |

不在 MVP 默认路径；作为可选增强。瓶颈在**区域编辑器 + 抠假透明**，不在 AI 出帧能力。

#### 分割 / 抠图（可选）

- 本地：`rembg`、`sam2`（若环境允许）
- 云端：各平台 segmentation API

### 交互编排运行时

**不**让 LLM 生成任意动画代码。采用：

```
模板 ID + 参数化序列 → Pixi 引擎执行
```

内置模板（初版）：`stagger_pop_in`、`tap_reveal`、`shake`、`float_idle`、`slide_in`、`scale_pulse` 等。

场景描述格式：JSON Scene Graph（字段约定见 `agreed-requirements.md`）。

### 阅读端（自研平板 App）

本仓库 **不实现**，仅预留：

- 场景 JSON 与资产包结构稳定后，可打包为 `.storypack`（zip）
- 平板端可用 Flutter / SwiftUI + 同类 2D 引擎复放场景图

## 非选型（MVP 明确不做）

| 项 | 原因 |
|----|------|
| 用户系统 / JWT / OAuth | 单用户 |
| PostgreSQL / Redis | 本地 SQLite 足够 |
| Kubernetes / Docker 部署 | 个人本地工具 |
| 微服务拆分 | 单体 + 清晰模块边界即可 |
| 自由画布编辑器（Fabric.js 全功能） | 固定版式 + 场景编排即可 |
| 全页视频管线 | 成本高，与交互目标冲突 |

## 开发与环境

| 项 | 建议 |
|----|------|
| 包管理 | pnpm monorepo（`apps/web` + `apps/server` + `packages/scene-schema`）或单仓前后端 |
| Node | 20 LTS |
| 类型共享 | `packages/scene-schema` 导出 Scene Graph、大纲、角色类型 |
| 代码规范 | ESLint + Prettier |
| 密钥 | `.env.local`，提供 `.env.example` |

## MVP 模块拆分（实现顺序建议）

```
1. packages/scene-schema     场景图、模板、角色类型定义
2. 本地存储 + 角色库 CRUD
3. LLM 梗概 → 大纲 + 对话修订
4. 生图适配器（1 个模型先跑通）+ 分层出图
5. Pixi 预览引擎 + L1 模板 1～2 个
6. 交互意图 → scene.json 生成
7. 页级资产区域编辑 + L2 帧序列预览
8. （后续）导出 .storypack
```

## 风险与对策

| 风险 | 对策 |
|------|------|
| 主角跨页不一致 | 角色库参考图 + 生成时强制注入描述；MVP 记录失败率 |
| 多模型接口差异大 | `ImageProvider` 适配器接口统一 |
| L2 帧间抖动 | 已初步可行；默认仍 L1，L2 作点缀；工具负责裁剪与对齐 |
| 对话改大纲破坏结构 | 大纲 schema 校验；按页 patch 而非全文重写 |
| Pixi 与 React 集成 | `@pixi/react` 或独立 canvas 挂载 |
