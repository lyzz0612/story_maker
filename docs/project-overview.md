# Story Maker — 项目概述

## 一句话

面向个人使用的 AI 互动绘本创作工具：用简略文字起稿，经对话打磨故事，为固定家庭主角生成跨书一致的形象，输出带翻页与页内交互的绘本；最终交付物供自研平板 App 阅读（导出格式后续定义）。

## 背景与动机

希望参考 BookletAI、ReadKidz 等产品的创作能力，但不做通用 SaaS。核心场景是**给自己孩子制作绘本**，单用户、本地项目为主。与竞品相比，本项目的差异化在于：

1. **家庭主角库**：宝宝、爸爸妈妈等固定角色在多本故事中形象一致；每本书的临时角色（如三只小猪）不要求跨书一致。
2. **对话式故事打磨**：输入极简梗概，AI 生成分页大纲后，在制作端通过对话调整细节。
3. **互动编排（主亮点）**：阅读体验不仅是翻页，还包括页内轻交互（例如介绍页点击后三只小猪依次冒出）。MVP 将验证当前 AI 在 L1（分层图 + 程序动效）乃至 L2（短序列帧）上的可行程度。

## 用户与场景

| 维度 | 说明 |
|------|------|
| 用户 | 开发者本人（兼创作者） |
| 读者 | 自己的孩子 |
| 使用方式 | 在**制作端**完成创作与预览；**阅读**在自研平板 App 中进行（依赖后续导出） |
| 多用户 | 初版不考虑账号、权限、协作、云同步 |

## 系统边界

```
┌──────────────────┐     导出（后续）     ┌──────────────────┐
│   Story Maker    │ ──────────────────▶ │  自研平板 App     │
│   （制作端）      │                     │  （阅读端）       │
│                  │                     │                  │
│ • 角色库         │                     │ • 翻页           │
│ • 故事 / 对话    │                     │ • 页内交互       │
│ • 配图 / 画风    │                     │ • （可选）配音   │
│ • 交互编排       │                     │                  │
│ • 本地预览       │                     │                  │
└──────────────────┘                     └──────────────────┘
```

- **制作端**：本仓库主要交付物。负责创作全流程与效果验证。
- **阅读端**：自研 App，不在本仓库初版范围内；导出契约后续单独设计。
- **不在初版范围**：多用户、KDP/YouTube 一键发布、自由画布编辑器、完整视频时间轴、网页公开分享。

## 核心用户旅程

```
① 维护家庭角色库（一次性 / 偶尔更新）
      建立宝宝、爸妈等标准形象

② 新建故事
      简略文字 + 勾选出场主角 + 选择画风与生图模型

③ AI 生成分页大纲
      每页摘要、出场角色（含临时角色）

④ 对话调整
      改情节、改单页、改语气；大纲面板与对话联动

⑤ 生成内容与交互
      逐页插图、排版、交互模板与元素编排

⑥ 制作端预览
      翻页 + 可点击的页内动效

⑦ 导出（后续）
      供平板 App 导入
```

## 功能模块（逻辑划分）

| 模块 | 职责 |
|------|------|
| **家庭角色库** | 固定主角档案、标准参考图、跨故事复用 |
| **故事工作台** | 简略输入、分页大纲、对话式修订 |
| **视觉引擎** | 画风预设、生图模型选择、分页出图；主角强一致、临时角色弱一致 |
| **交互编排** | 场景图层、动效模板、时序与触发器；主亮点 |
| **阅读预览** | 制作端内翻页与交互回放，用于验收 |
| **项目管理** | 单用户本地保存、项目列表、自动保存 |

## 互动体验分层

初版以验证 AI 能力为目标，交互能力分四级（详见 `agreed-requirements.md`）：

| 层级 | 内容 | MVP 态度 |
|------|------|----------|
| L0 | 翻页 + 静态整页图 | 兜底 |
| L1 | 分层资产 + 程序动效 + 模板库 | **默认主线** |
| L2 | 少量元素 3～4 帧循环动画 | **初步可行**（2026-06-08 探测）；点缀 + 需区域编辑工具 |
| L3 | 极少数页的短视频/循环视频层 | 非 MVP |

## MVP 目标

**目的**：用最小闭环测试当前 AI 在「固定主角一致」「分页故事」「分层配图」「互动编排（含可能的 L2）」上的实际表现，而非交付完整产品。

建议 MVP 闭环：

1. 创建 2～3 个家庭主角并锁定参考形象
2. 输入一条简略故事（如三只小猪，宝宝扮演小猪）
3. 生成并对话修订 6～8 页大纲
4. 选择一种画风 + 一种生图模型，生成配图
5. 至少 1 页实现 L1 交互（如 `stagger_pop_in`）
6. 至少 1 处 L2 循环动画（已初步验证可行，如小黑猪挥锤施工）
7. 页级资产区域编辑 + 制作端预览翻页与交互

**MVP 成功标准（探索性）**：

- 主角在多数页面上可辨认且跨页稳定
- 对话能可靠修改指定页大纲而不破坏结构
- L1 交互在预览中可复现且调整成本可接受
- L2：**已初步判定可行**；MVP 需跑通「图集导入 → 手框/裁剪 → 帧序列预览」工具链
